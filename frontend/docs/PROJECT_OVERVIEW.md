# 出摊啦 frontend 项目介绍

`frontend` 目录现在是基于 uni-app 的小程序/H5 前端项目

## 项目定位

本项目面向「出摊啦」流动摊位场景，核心目标是让顾客快速发现附近正在出摊的商户，让商户在手机端完成入驻、出摊和商品管理。

当前覆盖的端：

- 微信小程序：主要交付目标。
- uni-app H5：用于本地调试、页面预览和部分浏览器验证。

## 技术栈

- uni-app Vue 3
- Vite 5
- 原生能力：微信登录、定位、地图、上传图片、打开位置导航

## 主要功能

### 顾客端

- 商户列表：按当前位置查询附近出摊商户，支持关键词和分类筛选。
- 地图模式：使用小程序原生 `<map>` 展示商户点位。
- 商户详情：点击卡片或地图点位后拉取商户完整商品列表。
- 商品展示：列表卡片展示预览商品，详情弹窗展示完整商品。
- 收藏：收藏/取消收藏，收藏列表支持搜索和未出摊商户移除。
- 导航：地图详情页可调起微信位置导航。
- 分享入口：支持二维码分享,进入指定商户。
- 入驻入口：顾客可提交摊主入驻申请。

### 商户端

- 入驻申请状态：展示待审核、驳回、通过等状态。
- 申请资料维护：驳回或需补充时可修改并重新提交。
- 商户工作台：申请通过后进入商户端总览。
- 出摊管理：定位成功后可开始出摊，出摊照片作为工作台背景。
- 商品管理：新增、编辑、上下架、删除商品。
- 置顶商品：最多展示/维护 3 个置顶商品。
- 分享二维码

## 运行模式

`src/pages/customer/index.vue` 是当前唯一页面，内部按用户状态切换三种模式：

- `customer`：顾客列表/地图/收藏/详情。
- `application`：商户入驻申请状态和资料填写。
- `merchant`：商户工作台。

页面模式由后端状态接口和本地交互共同决定。用户提交入驻申请后再次打开会进入申请状态页；审核通过后会进入商户工作台，同时仍保留顾客/商户切换入口。

## 接口分组

项目代码按后端路由分组调用：

- `/api/pub/*`：公开查询类接口，如附近商户、分享信息、公开商品。
- `/api/customer/*`：顾客登录后接口，如收藏、入驻申请、反馈。
- `/api/merchant/*`：商户能力接口，如商户资料、商品、出摊。
- `/api/Wx/mini`：小程序登录，使用 `uni.login` code 换取 token。
